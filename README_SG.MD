## Getting Started With the Task List 

This document explains how project contributors should use the task list in GitHub to manage work.

### 1. Open the Task List
1. Open the project on GitHub.
2. Click on the **Issues** tab at the top.
3. This is where all tasks for the project are stored.

### 2. View Current Tasks
- Each issue represents one task.
- The issue **title** tells you what the task is.
- The **description** explains what needs to be done.
- Open issues = tasks not finished  
- Closed issues = completed tasks

### 3. Add a New Task
1. Click **New issue**.
2. In **Title**, write a short task name.  
   Example: `Add search bar`
3. In **Description**, explain what needs to be completed.
4. If the group is using milestones, select the correct one.  
   Example: `Studentname_v1.0 Release`
5. Click **Submit new issue**.

### 4. Work on a Task
1. Click the issue you are working on.
2. Read the description carefully.
3. Make changes to the project on your computer.

### 5. Complete and Close a Task
1. After finishing the task, go back to the issue on GitHub.
2. Write a comment stating what you completed.
3. Click **Close issue** to mark it as done.
